# Repository Notes

## Shape
- Keep this file up to date when future sessions learn important repo-specific behavior, but only add guidance an agent would likely miss.
- This is a dotfiles/Home Manager flake, not an app. `home/`, `opencode/`, and `repos/` are Stow package roots laid out as `$HOME` contents.
- `mksymlinks.sh` is the native Stow entrypoint. With no args it restows `home` and `opencode`; extra args are appended, so use `bash mksymlinks.sh repos` to include repo tooling.
- `dotfiles.nix` owns deployment mechanics. Default Home Manager mode is `out-of-store`, default packages are `home`, `repos`, and `opencode`.
- `common.nix` is shared Home Manager config; `t14.nix` is the `gsr` machine/user layer using external `cli-tools`/`mypkgs` packages.
- Fish source snippets live under `fish/` and are wired into Home Manager from `common.nix`.

## Deployment Gotchas
- Home Manager `out-of-store` requires a checkout at `~/terminalenv` unless `dotfiles.links.repoRoot` is overridden; activation intentionally fails without it.
- Home Manager `store` mode materializes Stow package files into the Nix store and is what external flake consumers can use without a checkout.
- `home/.config/nvim/init.vim` is the user config; Home Manager generates `~/.config/nvim/hm-generated.lua` and `common.nix` force-disables generated `init.lua`.
- Bash files are stowed from `home/.bashrc` and `home/.config/bash/`; zsh and fish config are primarily generated by Home Manager in `common.nix`.
- The repo-local `.envrc` is intentional: it adds `scripts/direnv/bin` so the local `tree` wrapper can affect commands run from this repo.

## Style
- When embedded file contents or scripts in Nix become long, move them into separate files instead of growing large inline strings.
- Commit local work in logical bundles when appropriate, but never push from an agent session.

## Commands
- Fast evaluation: `nix eval .#homeConfigurations.gsr.activationPackage.drvPath`.
- Main in-repo check: `nix flake check`.
- Focused deployment check: `nix build "path:$PWD#checks.x86_64-linux.deployment-tests"`.
- Full container integration test requires the warmed Podman image `localhost/terminalenv-tests:latest`: run `bash tests/refresh-test-image.sh` first, then `bash tests/test-deployments.sh`.
- Add `-v` to `tests/refresh-test-image.sh` or `tests/test-deployments.sh` for full logs; default output is intentionally concise.

## Test Coverage
- `tests/test-deployments-in-nix-check.sh` is the lightweight flake check path and validates native symlinks plus Home Manager generation output.
- `tests/verify-deployment.sh` is the full container scenario runner: native Stow, Home Manager `out-of-store`, Home Manager `store`, and external flake consumer behavior.
